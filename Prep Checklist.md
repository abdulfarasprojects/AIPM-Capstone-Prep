## Reusable Demo-Day Prep Checklist

### 1. Problem
- [ ] State the user in one sentence
- [ ] State the pain in one sentence
- [ ] Explain why this problem matters now
- [ ] Show current workaround
- [ ] Quantify cost of the problem if possible

### 2. ICP
- [ ] Name the exact target user
- [ ] Name the buyer if different
- [ ] Narrow the segment enough to feel specific
- [ ] Be ready to explain why this ICP is the right starting point

### 3. Discovery
- [ ] List how many users/customers you spoke with
- [ ] Summarize the top repeated pain points
- [ ] Note what surprised you
- [ ] Be ready to explain what changed in your product because of discovery

### 4. Product Wedge
- [ ] Identify the single best entry feature
- [ ] Be ready to explain why you started there
- [ ] Know what you would cut for MVP
- [ ] Make sure the scope feels ambitious but focused

### 5. Workflow Fit
- [ ] Show where your product fits in the user journey
- [ ] Show what happens before the AI step
- [ ] Show what happens after the AI output
- [ ] Make sure the AI feels integrated, not bolted on
- [ ] Be ready to explain onboarding

### 6. Why AI
- [ ] Explain why AI is necessary here
- [ ] Explain why rules/manual workflows are insufficient
- [ ] Clarify what task the model is doing
- [ ] Separate what AI does vs what deterministic systems do

### 7. Demo Quality
- [ ] Start with a relatable user story
- [ ] Show the key value moment early
- [ ] Keep the demo path simple
- [ ] Avoid too many screens or too much text
- [ ] Make sure the demo clearly shows differentiation
- [ ] Practice pacing so you don’t rush

### 8. Evals
- [ ] Define what “good output” means
- [ ] List your eval dimensions
- [ ] Test happy paths
- [ ] Test failure paths
- [ ] Test edge cases
- [ ] Know what happens when confidence is low
- [ ] Be ready to explain human-in-the-loop points

### 9. Metrics
- [ ] Pick 1–2 core success metrics
- [ ] Prefer outcome metrics over vanity metrics
- [ ] Know how success will be measured in production
- [ ] Be ready to explain what metric proves customer value

### 10. Safety / Reliability
- [ ] Identify biggest failure modes
- [ ] Explain fallback behavior
- [ ] Explain escalation to humans if relevant
- [ ] Prepare answers on consistency, hallucinations, and abuse
- [ ] If relevant, prepare for security / compliance / privacy questions

### 11. Differentiation
- [ ] Be ready to answer “why not just use ChatGPT?”
- [ ] Be ready to answer “why can’t incumbents copy this?”
- [ ] Explain your moat clearly
- [ ] Clarify what is uniquely vertical or workflow-specific

### 12. GTM
- [ ] Identify first target market
- [ ] Name the initial buyer
- [ ] Explain how you’ll get first users/pilots
- [ ] Prepare a simple adoption motion
- [ ] Know likely objections and how you’ll respond

### 13. Monetization
- [ ] Have a pricing model
- [ ] Be ready to explain why that pricing fits value
- [ ] Know approximate cost to serve
- [ ] Be ready to explain ROI

### 14. Roadmap
- [ ] Know what comes after the MVP
- [ ] Be ready to explain what you’re not doing yet
- [ ] Have a clear next-step roadmap
- [ ] Show expansion logic from wedge to broader platform

### 15. Presentation Readiness
- [ ] One idea per slide
- [ ] Keep slides visually simple
- [ ] Avoid dense text
- [ ] Prepare a strong opening and closing
- [ ] Rehearse answers to likely judge questions
- [ ] Time the full presentation and Q&A

## 10 must-have answers before demo day
- [ ] Who is this for?
- [ ] What painful problem does it solve?
- [ ] Why now?
- [ ] Why AI?
- [ ] Why this wedge first?
- [ ] How does it fit into workflow?
- [ ] How do you know it works?
- [ ] What happens when it fails?
- [ ] Why will people pay?
- [ ] Why will this win?
